## 7. Примеры замены (до → после)

### Пример 1: Задачи

```
БЫЛО (115 символов - слишком длинно):
"Разрабатывать архитектурные разделы проектов (концепции, П, Р, рабочая документация)
в соответствии с техническим заданием и требованиями САО/РП."

СТАЛО (52 символа - компактно):
"Разработка архитектурных разделов проектов"

или

"Подготовка проектной и рабочей документации"

ПРИНЦИП:
- Убрать детали (саму отнесет в metrics/KPI)
- Оставить суть (что делать)
- Формат: [Действие] + [Объект]
```

### Пример 2: Навыки (инструменты)

```
БЫЛО (инструмент-ориентированный):
"Revit/BIM — моделирование архитектурных разделов, создание экспортов
для Clash detection и координации" (102 символа)

СТАЛО (компетенция-ориентированный):
"Умение выполнять моделирование архитектурных объектов" (53 символа)

или

"Умение работать с системами BIM-моделирования" (50 символов)

ПРИНЦИП:
- Указывать класс инструмента, не конкретный продукт
- Фокус на КОМПЕТЕНЦИИ, не на ИНСТРУМЕНТ
- Инструменты меняются (Revit → новая система), компетенция остается
```

### Пример 3: Уровни

```
БЫЛО (одинаково для всех):
Level 2: "Существенные знания и опыт применения знаний в ситуациях
         повышенной сложности, в т.ч. в кризисных ситуациях"
Level 3: "Существенные знания и опыт применения знаний в ситуациях
         повышенной сложности, в т.ч. в кризисных ситуациях"

СТАЛО (разные для каждого):
Level 1: "Базовые знания и опыт применения в стандартных ситуациях"
Level 2: "Существенные знания и опыт применения в ситуациях повышенной сложности"
Level 3: "Глубокие знания, опыт применения в кризисных ситуациях, готовность обучать других"

ПРИНЦИП:
- Каждый уровень должен быть ЧЕТКО отличим от остальных
- Для каждого уровня - свое описание
- Описание должно отражать реальную разницу в компетенции
```

```

  [ ] Максимальная длина <= 100 символов

  [ ] Категория "Управление временем" присутствует
  [ ] Категория "Коммуникация" присутствует
  [ ] Нет аббревиатур
  [ ] Нет конкретных инструментов (кроме класса типа "САПР")

  [ ] Используются уровни 1, 2, 3
  [ ] У каждого уровня СВОЁ описание
  [ ] Описания четко отличаются друг от друга
```
